# 自定义组件开发指南

本文档描述如何在 go-data-flow 框架中开发和使用自定义组件。

## 1. 项目结构

```
go-data-flow/
├── cmd/dataflow/main.go      # 标准入口（注册全部内置组件）
├── pkg/dataflow/
│   ├── app/                  # 应用框架
│   │   ├── app.go           # App 核心逻辑
│   │   └── builtins.go      # 内置组件注册函数
│   └── builtins/            # 内置组件实现
│       ├── source/          # Source 组件
│       ├── processor/       # Processor 组件
│       └── sink/            # Sink 组件
├── examples/config/          # 配置示例（开发调试用）
└── dist/                     # 编译输出目录
```

## 2. 组件接口

### 2.1 Source 接口

```go
type Source[T any] interface {
    // Init 初始化（只解析配置，不建立连接）
    Init(config []byte) error

    // Read 读取数据并发送到输出通道
    Read(ctx context.Context, out chan<- T) (int64, error)
}
```

### 2.2 Processor 接口

```go
type Processor[T any] interface {
    // Init 初始化
    Init(config []byte) error

    // Process 处理数据
    Process(ctx context.Context, in <-chan T, out chan<- T) error
}
```

### 2.3 Sink 接口

```go
type Sink[T any] interface {
    // Init 初始化
    Init(config []byte) error

    // Consume 消费数据
    Consume(ctx context.Context, in <-chan T) error
}
```

## 3. 开发自定义组件

### 3.1 创建组件文件

在 `internal/components/` 下创建组件（租户项目结构参见[租户二次开发指南](02_租户二次开发指南.md)）：

```go
// internal/components/source/reader.go
package source

import (
    "context"

    "github.com/Hopetree/go-data-flow/pkg/dataflow"
    "github.com/Hopetree/go-data-flow/pkg/dataflow/builtins/types"
)

// TypeName 组件注册名称（在 Flow YAML 的 name 字段中使用）
const TypeName = "source-my-data"

// Config 配置结构
type Config struct {
    Host string `json:"host"`
    Port int    `json:"port"`
}

// MySource 自定义 Source
type MySource struct {
    config *Config
}

// New 构造函数
func New() *MySource {
    return &MySource{}
}

// Init 初始化
func (s *MySource) Init(config []byte) error {
    // 解析配置
    return nil
}

// Read 读取数据并发送到输出通道
func (s *MySource) Read(ctx context.Context, out chan<- types.Record) (int64, error) {
    var count int64

    // 建立连接、读取数据
    for {
        select {
        case <-ctx.Done():
            return count, ctx.Err()
        case out <- types.Record{"key": "value"}:
            count++
        }
    }
}

// Register 注册组件到 Registry
// 建议在每个组件文件中实现 Register 函数，并在包级别定义 TypeName，
// 这样租户在 main.go 中只需调用各包的 Register 即可。
func Register(r *dataflow.Registry[types.Record]) {
    r.RegisterSource(TypeName, func() dataflow.Source[types.Record] {
        return New()
    })
}

// 编译时接口检查
var _ dataflow.Source[types.Record] = (*MySource)(nil)
```

### 3.2 在 main.go 中注册

组件文件中已提供 `Register` 函数，main.go 只需调用即可（完整 main.go 参见[租户二次开发指南](02_租户二次开发指南.md)）：

```go
import (
    "github.com/Hopetree/go-data-flow/pkg/dataflow/app"
    "my-dataflow/internal/components/source"
)

func main() {
    application := app.NewApp(app.Options{...})

    // 注册内置组件
    app.RegisterAllBuiltins(application.Registry())

    // 注册自定义组件（每个组件包提供 Register 函数）
    source.Register(application.Registry())

    application.Run(ctx)
}
```

## 4. 注册策略

### 4.1 使用全部内置组件

```go
app.RegisterAllBuiltins(application.Registry())
```

### 4.2 覆盖内置组件

```go
// 先注册内置
app.RegisterAllBuiltins(application.Registry())

// 再注册自定义（同名覆盖）
r.RegisterSource("source-static-data", func() dataflow.Source[types.Record] {
    return &MyCustomSource{}
})
```

## 5. 配置文件使用

注册后即可在 YAML 配置中使用：

```yaml
name: my-flow
source:
  name: source-my-data    # 使用自定义组件
  config:
    host: localhost
    port: 3306
processors:
  - name: processor-condition-filter
    config:
      field: status
      op: eq
      value: 200
sink:
  name: sink-output-console
```

## 7. 自定义指标

框架内置了 Prometheus 指标收集功能。对于大多数组件，框架的自动收集已经足够。如果组件需要记录额外的业务指标（如过滤丢弃数、HTTP 状态码分布等），可以实现可选接口来获取指标记录器。

### 7.1 框架自动收集（无需额外代码）

框架自动为每个组件记录以下基础指标，无需组件做任何事：

| 指标名 | 类型 | 说明 |
|--------|------|------|
| `records_in_total` | Counter | 进入组件的记录数 |
| `records_out_total` | Counter | 离开组件的记录数 |
| `records_error_total` | Counter | 组件错误数 |
| `duration_seconds` | Histogram | 处理耗时（秒） |

启用方式——在 `app.yaml` 中配置即可，无需修改组件代码：

```yaml
metrics:
  enabled: true         # 启用指标服务
  addr: ":9090"         # 监听地址
  path: "/metrics"      # 指标路径
  namespace: "dataflow"  # 指标命名空间
```

启用后可通过 `http://localhost:9090/metrics` 访问 Prometheus 指标。

### 7.2 什么时候需要自定义指标

如果组件需要跟踪框架基础指标无法覆盖的业务数据，就需要自定义指标。典型场景：

- **过滤器**：记录被丢弃的记录数、按条件分组统计
- **HTTP Source**：记录请求状态码分布
- **聚合器**：记录每个分组的记录数
- **外部调用**：记录调用量、延迟、失败数

如果只是需要输入/输出/错误/耗时统计，直接使用框架自动收集即可，不需要阅读后续内容。

### 7.3 相关接口定义

自定义指标只需要实现一个可选接口。以下接口均定义在框架源码中，组件按需实现：

**MetricsRecorderAware**（`pkg/dataflow/types.go`）—— 组件实现此接口接收框架注入的 Recorder：

```go
type MetricsRecorderAware interface {
    SetMetricsRecorder(recorder metrics.Recorder, info ComponentInfo)
}

// ComponentInfo 注入时提供的组件上下文
type ComponentInfo struct {
    FlowName      string  // Flow 名称（来自 YAML 的 name 字段）
    ComponentName string  // 组件标识，如 "source"、"processor[0]"、"sink"
    ComponentType string  // 组件类型：source / processor / sink
}
```

**Recorder**（`pkg/dataflow/metrics/recorder.go`）—— 框架注入给组件的指标记录器：

```go
type Recorder interface {
    // 基础指标（框架自动调用，组件不需要手动调用）
    RecordIn(count int64)
    RecordOut(count int64)
    RecordError(count int64)
    RecordDuration(seconds float64)

    // 自定义指标（组件手动调用）
    AddCounter(name string, value float64, labels prometheus.Labels)
    SetGauge(name string, value float64, labels prometheus.Labels)
    ObserveHistogram(name string, value float64, labels prometheus.Labels)
    ObserveSummary(name string, value float64, labels prometheus.Labels)
}
```

框架在组件 `Init()` 成功后自动检测是否实现了 `MetricsRecorderAware`，如果是则注入 `Recorder`。不实现此接口的组件不受任何影响。

### 7.4 实现步骤

只需要两步：

#### 步骤 1：在结构体中持有 Recorder，实现 SetMetricsRecorder

```go
import (
    "github.com/Hopetree/go-data-flow/pkg/dataflow"
    "github.com/Hopetree/go-data-flow/pkg/dataflow/metrics"
)

type Processor struct {
    config   *Config
    recorder metrics.Recorder // 框架注入
}

// SetMetricsRecorder 框架在 Init() 之后自动调用
func (p *Processor) SetMetricsRecorder(recorder metrics.Recorder, info dataflow.ComponentInfo) {
    p.recorder = recorder
}
```

#### 步骤 2：在 Read/Process/Consume 中调用 Recorder 方法

```go
func (p *Processor) Process(ctx context.Context, in <-chan types.Record, out chan<- types.Record) error {
    for record := range in {
        if p.shouldDrop(record) {
            // 记录自定义业务指标
            p.recorder.AddCounter("dropped_total", 1, prometheus.Labels{
                "field": p.config.Field,
            })
            continue
        }
        out <- record
    }
    return nil
}
```

### 7.5 自定义指标类型

| 类型 | 方法 | 用途 | 示例 |
|------|------|------|------|
| Counter | `AddCounter(name, value, labels)` | 累计计数，只增不减 | 过滤丢弃数、请求数 |
| Gauge | `SetGauge(name, value, labels)` | 可增可减的当前值 | 当前队列长度、连接数 |
| Histogram | `ObserveHistogram(name, value, labels)` | 分布统计 | 请求延迟、响应大小 |
| Summary | `ObserveSummary(name, value, labels)` | 分位数统计 | 响应时间的 P99 |

### 7.6 Labels 使用说明

框架会自动为所有指标附加 `flow` 和 `component` 标签，组件只需传入**业务相关的标签**：

```go
// 组件只传业务标签
p.recorder.AddCounter("http_requests_total", 1, prometheus.Labels{
    "status": "200",
    "method": "GET",
})

// Prometheus 中实际生成的指标：
// dataflow_http_requests_total{flow="my-flow", component="source", status="200", method="GET"}
```

### 7.7 完整示例：带自定义指标的过滤器

```go
// internal/components/processor/condition_filter.go
package condition_filter

import (
    "context"
    "encoding/json"
    "fmt"

    "github.com/prometheus/client_golang/prometheus"

    "github.com/Hopetree/go-data-flow/pkg/dataflow"
    "github.com/Hopetree/go-data-flow/pkg/dataflow/builtins/types"
    "github.com/Hopetree/go-data-flow/pkg/dataflow/metrics"
)

const TypeName = "processor-condition-filter"

type Config struct {
    Field    string      `json:"field"`
    Operator string      `json:"operator"`
    Value    interface{} `json:"value"`
}

type Processor struct {
    config   *Config
    recorder metrics.Recorder // 可选：框架注入（不使用自定义指标时可为 nil）
}

func New() *Processor {
    return &Processor{
        config: &Config{},
    }
}

func (p *Processor) Init(config []byte) error {
    if err := json.Unmarshal(config, p.config); err != nil {
        return fmt.Errorf("parse config failed: %w", err)
    }
    if p.config.Field == "" {
        return fmt.Errorf("field is required")
    }
    if p.config.Operator == "" {
        p.config.Operator = "eq"
    }
    return nil
}

// SetMetricsRecorder 可选：框架在 Init() 成功后自动调用。
// 如果不需要自定义指标，删除此方法即可，框架的自动收集不受影响。
func (p *Processor) SetMetricsRecorder(recorder metrics.Recorder, info dataflow.ComponentInfo) {
    p.recorder = recorder
}

func (p *Processor) Process(ctx context.Context, in <-chan types.Record, out chan<- types.Record) error {
    baseLabels := prometheus.Labels{
        "field":    p.config.Field,
        "operator": p.config.Operator,
    }

    for record := range in {
        select {
        case <-ctx.Done():
            return ctx.Err()
        default:
        }

        if p.match(record) {
            p.recorder.AddCounter("filter_passed_total", 1, baseLabels)
            out <- record
        } else {
            p.recorder.AddCounter("filter_dropped_total", 1, baseLabels)
        }
    }
    return nil
}

func (p *Processor) match(record types.Record) bool {
    value, ok := record[p.config.Field]
    if !ok {
        return false
    }
    switch p.config.Operator {
    case "eq":
        return fmt.Sprintf("%v", value) == fmt.Sprintf("%v", p.config.Value)
    case "ne":
        return fmt.Sprintf("%v", value) != fmt.Sprintf("%v", p.config.Value)
    default:
        return false
    }
}

func Register(r *dataflow.Registry[types.Record]) {
    r.RegisterProcessor(TypeName, func() dataflow.Processor[types.Record] {
        return New()
    })
}
```

> **关键点**：自定义指标功能完全可选。如果不需要业务指标，只保留 `New()`、`Init()`、`Process()` 和 `Register()` 即可，框架的自动收集仍然正常工作。

### 7.8 Prometheus 查询示例

```promql
# 查询某个 Flow 的输入记录数（框架自动收集）
dataflow_records_in_total{flow="my-flow"}

# 查询自定义指标：被丢弃的记录数
dataflow_filter_dropped_total{flow="my-flow"}

# 查询某个组件的处理耗时分布
histogram_quantile(0.95, rate(dataflow_duration_seconds_bucket{flow="my-flow", component="processor[0]"}[5m]))
```

## 8. 开发清单

开发自定义组件时，确保：

- [ ] 定义配置结构体，包含 JSON 标签
- [ ] 配置有合理的默认值
- [ ] Init 中校验必填字段
- [ ] Read/Process/Consume 支持 ctx 取消
- [ ] 使用 select 防止通道阻塞
- [ ] 错误信息包含上下文
- [ ] 定义 `TypeName` 常量和 `Register` 函数
- [ ] 在 main.go 中调用各组件包的 `Register`
- [ ] （可选）实现 `MetricsRecorderAware` 接口，使用 `Recorder` 记录自定义业务指标
- [ ] （可选）在 app.yaml 中启用 metrics 服务进行监控
