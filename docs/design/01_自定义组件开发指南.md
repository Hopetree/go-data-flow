# 自定义组件开发指南

本文档描述如何在 go-data-flow 框架中开发和使用自定义组件。

## 1. 项目结构

```
go-data-flow/
├── cmd/dataflow/main.go      # 标准入口（注册全部内置组件）
├── pkg/dataflow/
│   ├── app/                  # 应用框架
│   │   ├── app.go           # App 核心逻辑
│   │   └── builtins.go      # 内置组件注册函数
│   └── builtins/            # 内置组件实现
│       ├── source/          # Source 组件
│       ├── processor/       # Processor 组件
│       └── sink/            # Sink 组件
├── examples/config/          # 配置示例（开发调试用）
└── dist/                     # 编译输出目录
```

## 2. 组件接口

### 2.1 Source 接口

```go
type Source[T any] interface {
    // Init 初始化（只解析配置，不建立连接）
    Init(config []byte) error

    // Read 读取数据并发送到输出通道
    Read(ctx context.Context, out chan<- T) (int64, error)
}
```

### 2.2 Processor 接口

```go
type Processor[T any] interface {
    // Init 初始化
    Init(config []byte) error

    // Process 处理数据
    Process(ctx context.Context, in <-chan T, out chan<- T) error
}
```

### 2.3 Sink 接口

```go
type Sink[T any] interface {
    // Init 初始化
    Init(config []byte) error

    // Consume 消费数据
    Consume(ctx context.Context, in <-chan T) error
}
```

## 3. 开发自定义组件

### 3.1 创建组件文件

在 `pkg/dataflow/builtins/` 下创建组件，或在自己的包中创建：

```go
// my_source.go
package mypackage

import (
    "context"
    "encoding/json"
    "fmt"

    "github.com/yourorg/go-data-flow/pkg/dataflow/app"
)

// MySourceConfig 配置结构
type MySourceConfig struct {
    Host string `json:"host"`
    Port int    `json:"port"`
}

// MySource 自定义 Source
type MySource struct {
    config *MySourceConfig
}

// NewMySource 构造函数
func NewMySource() *MySource {
    return &MySource{}
}

// Init 初始化
func (s *MySource) Init(config []byte) error {
    cfg := &MySourceConfig{Port: 3306} // 默认值
    if err := json.Unmarshal(config, cfg); err != nil {
        return fmt.Errorf("解析配置失败: %w", err)
    }
    if cfg.Host == "" {
        return fmt.Errorf("host 不能为空")
    }
    s.config = cfg
    return nil
}

// Read 读取数据
func (s *MySource) Read(ctx context.Context, out chan<- app.Record) (int64, error) {
    var count int64

    // 建立连接、读取数据
    for {
        select {
        case <-ctx.Done():
            return count, ctx.Err()
        case out <- app.Record{"key": "value"}:
            count++
        }
    }
}
```

### 3.2 注册组件

在 main.go 中注册：

```go
package main

import (
    "github.com/yourorg/go-data-flow/pkg/dataflow"
    "github.com/yourorg/go-data-flow/pkg/dataflow/app"
)

func registerComponents(r *dataflow.Registry[app.Record]) {
    // 1. 按需注册内置组件
    app.RegisterBuiltinSources(r)
    app.RegisterBuiltinProcessors(r)
    app.RegisterBuiltinSinks(r)

    // 2. 注册自定义组件（同名会覆盖内置）
    r.RegisterSource("my-source", func() dataflow.Source[app.Record] {
        return NewMySource()
    })
}

func main() {
    // ...

    application := app.NewApp(app.Options{...})
    registerComponents(application.Registry())  // 注册组件

    application.Run(ctx)
}
```

## 4. 注册策略

### 4.1 使用全部内置组件

```go
app.RegisterAllBuiltins(application.Registry())
```

### 4.2 按需注册内置组件

```go
// 只注册需要的类型
app.RegisterBuiltinSources(r)      // Source: console, csv, json, generator
app.RegisterBuiltinProcessors(r)   // Processor: filter, transform, aggregate
app.RegisterBuiltinSinks(r)        // Sink: console, csv, json, file, null, collect
```

### 4.3 覆盖内置组件

```go
// 先注册内置
app.RegisterBuiltinSources(r)

// 再注册自定义（同名覆盖）
r.RegisterSource("console", func() dataflow.Source[app.Record] {
    return &MyCustomConsoleSource{}
})
```

## 5. 配置文件使用

注册后即可在 YAML 配置中使用：

```yaml
name: my-flow
source:
  name: my-source      # 使用自定义组件
  config:
    host: localhost
    port: 3306
processors:
  - name: filter
    config:
      field: status
      op: eq
      value: 200
sink:
  name: console
```

## 7. 指标收集

框架内置了 Prometheus 指标收集功能，自动为每个组件记录以下指标：

| 指标名 | 类型 | 说明 |
|--------|------|------|
| `records_in_total` | Counter | 进入组件的记录数 |
| `records_out_total` | Counter | 离开组件的记录数 |
| `records_error_total` | Counter | 组件错误数 |
| `duration_seconds` | Histogram | 处理耗时（秒） |

### 7.1 启用指标服务

在应用配置文件（如 `app.yaml`）中启用 Prometheus 指标服务：

```yaml
name: dataflow
log:
  level: info
  format: text
metrics:
  enabled: true        # 启用指标服务
  addr: ":9090"        # 监听地址
  path: "/metrics"     # 指标路径
  namespace: "dataflow" # 指标命名空间
```

启用后，可通过 `http://localhost:9090/metrics` 访问指标。

### 7.2 框架自动收集

框架在运行时自动为每个组件收集指标：

- **Source**: 自动记录输出记录数（`RecordOut`）
- **Processor**: 自动记录输入/输出记录数和处理耗时
- **Sink**: 自动记录输入记录数和处理耗时

所有组件出错时都会自动记录错误数。

### 7.3 指标查看

运行完成后，框架会打印指标汇总：

```
========== [my-flow] 指标汇总 ==========
运行时间: 1.5s
总输入: 1000
总输出: 950
总错误: 5
--- 组件指标 ---
  [source] 输入=0, 输出=1000, 错误=0, 平均耗时=0.0000s
  [processor[0]] 输入=1000, 输出=980, 错误=0, 平均耗时=0.0012s
  [sink] 输入=950, 输出=0, 错误=0, 平均耗时=0.0005s
```

### 7.4 指标接口

框架定义了以下指标相关接口：

#### MetricsCapable（声明自定义指标）

组件可实现此接口来声明需要暴露的自定义指标：

```go
// MetricType 指标类型
type MetricType int

const (
    MetricTypeCounter   MetricType = iota // 计数器，只增不减
    MetricTypeGauge                      // 可增可减的值
    MetricTypeHistogram                  // 直方图，分布统计
    MetricTypeSummary                    // 摘要，计算分位数
)

// MetricDesc 指标描述
type MetricDesc struct {
    Name   string      // 指标名称（不含命名空间前缀）
    Type   MetricType  // 指标类型
    Help   string      // 帮助文本
    Labels []string    // 标签名称列表
}

// MetricsCapable 声明自定义指标接口
type MetricsCapable interface {
    Metrics() []MetricDesc
}
```

> **重要**： 组件必须先实现 `MetricsCapable` 接口声明指标，才能在代码中使用 `AddCounter` 等方法。

#### MetricsRecorder（指标记录器）

```go
// MetricsRecorder 记录处理过程中的指标
type MetricsRecorder interface {
    // 基础指标（框架自动调用）
    RecordIn(count int64)        // 记录进入组件的记录数
    RecordOut(count int64)       // 记录离开组件的记录数
    RecordError(count int64)     // 记录错误数
    RecordDuration(seconds float64) // 记录处理耗时

    // 自定义指标（组件内部调用）
    AddCounter(name string, value float64, labels prometheus.Labels)
    SetGauge(name string, value float64, labels prometheus.Labels)
    ObserveHistogram(name string, value float64, labels prometheus.Labels)
}
```

框架自动为基础指标调用 `RecordIn`/`RecordOut`/`RecordError`/`RecordDuration`。
组件可通过 `AddCounter`/`SetGauge`/`ObserveHistogram` 记录自定义业务指标。

#### MetricsRecorderAware（注入指标记录器）

```go
// MetricsRecorderAware 组件可实现的接口，用于获取指标记录器
type MetricsRecorderAware interface {
    SetMetricsRecorder(recorder MetricsRecorder, info ComponentInfo)
}

// ComponentInfo 组件上下文信息
type ComponentInfo struct {
    FlowName      string  // Flow 名称
    ComponentName string  // 组件名称（如 "processor[0]"）
    ComponentType string  // 组件类型：source/processor/sink
}
```

组件实现此接口后，框架会在 `Init` 之后自动注入 `MetricsRecorder`。

### 7.5 在组件中记录自定义指标

如果组件需要记录自定义业务指标（如过滤丢弃的数据量、HTTP 状态码分布等），需要完成以下步骤：

#### 步骤1：声明自定义指标

组件必须实现 `MetricsCapable` 接口来声明需要暴露的指标，框架会在构建时自动注册：

```go
// FilterProcessor 带自定义指标的过滤器
type FilterProcessor struct {
    config   *FilterConfig
    recorder dataflow.MetricsRecorder  // 步骤2: 框架注入
    info     dataflow.ComponentInfo       // 步骤2: 框架注入
}

// Metrics 声明自定义指标（必须先实现这个！）
func (p *FilterProcessor) Metrics() []dataflow.MetricDesc {
    return []dataflow.MetricDesc{
        {
            Name:   "filter_dropped_total",
            Type:   dataflow.MetricTypeCounter,
            Help:   "Total number of records dropped by filter",
            Labels: []string{"flow", "component", "condition"},
        },
    }
}
```

> **重要**： 必须先声明指标才能在代码中使用 `AddCounter` 等方法，否则指标不会被 Prometheus 收集。

#### 步骤2：实现 MetricsRecorderAware 接口

框架会通过此接口将 `MetricsRecorder` 注入到组件中：

```go
// SetMetricsRecorder 框架调用，注入指标记录器
func (p *FilterProcessor) SetMetricsRecorder(recorder dataflow.MetricsRecorder, info dataflow.ComponentInfo) {
    p.recorder = recorder
    p.info = info
}
```

#### 步骤3：在 Process/Read/Consume 中使用指标

```go
// Process 处理数据
func (p *FilterProcessor) Process(ctx context.Context, in <-chan app.Record, out chan<- app.Record) error {
    for record := range in {
        if p.shouldFilter(record) {
            // 记录自定义业务指标：过滤丢弃的数据量
            p.recorder.AddCounter("filter_dropped_total", 1, prometheus.Labels{
                "condition": p.config.Condition,
            })
            continue
        }
        out <- record
    }
    return nil
}

func (p *FilterProcessor) shouldFilter(record app.Record) bool {
    // 过滤逻辑...
    return false
}
```

> **注意**： `AddCounter` 的 `labels` 参数只需要传入业务相关的标签（如 `condition`），框架会自动附加 `flow` 和 `component` 标签。

### 7.6 匇标类型

| 类型 | 方法 | 用途 |
|------|------|------|
| Counter | `AddCounter(name, value, labels)` | 累计计数器，只增不减（如请求数、错误数） |
| Gauge | `SetGauge(name, value, labels)` | 可增可减的值（如当前连接数、队列长度） |
| Histogram | `ObserveHistogram(name, value, labels)` | 分布统计（如请求延迟、响应大小） |

### 7.7 Prometheus 标签

所有指标自动包含以下标签：

| 标签 | 说明 |
|------|------|
| `flow` | Flow 名称（来自配置文件的 `name` 字段） |
| `component` | 组件名称（如 `source`、`processor[0]`、`sink`） |

在 Grafana 中查询示例：

```promql
# 查询某个 Flow 的输入记录数
dataflow_records_in_total{flow="my-flow"}

# 查询某个组件的处理耗时分布
histogram_quantile(0.95, rate(dataflow_duration_seconds_bucket{flow="my-flow", component="processor[0]"}[5m]))
```

## 8. 完整示例：带自定义指标的过滤器组件

本节演示如何开发一个完整的过滤器组件，包含两个自定义指标：
- `filter_passed_total`: 通过过滤的记录数
- `filter_dropped_total`: 被过滤丢弃的记录数（按条件分组）

### 8.1 目录结构

```
pkg/dataflow/builtins/processor/
└── condition_filter/
    └── condition_filter.go    # 过滤器组件实现
```

### 8.2 组件实现

**文件**: `pkg/dataflow/builtins/processor/condition_filter/condition_filter.go`

```go
// Package condition_filter 提供带自定义指标的过滤处理器
package condition_filter

import (
	"context"
	"encoding/json"
	"fmt"

	"github.com/prometheus/client_golang/prometheus"

	"github.com/yourorg/go-data-flow/pkg/dataflow"
	"github.com/yourorg/go-data-flow/pkg/dataflow/builtins/types"
)

// Config 过滤器配置
type Config struct {
	// Field 要过滤的字段名
	Field string `json:"field"`
	// Operator 比较操作符: eq, ne, gt, lt, gte, lte, contains
	Operator string `json:"operator"`
	// Value 比较值
	Value interface{} `json:"value"`
}

// Processor 带自定义指标的过滤处理器
type Processor struct {
	config   *Config
	recorder dataflow.MetricsRecorder // 框架注入
	info     dataflow.ComponentInfo   // 框架注入
}

// New 创建新的过滤器实例
func New() *Processor {
	return &Processor{
		config: &Config{},
	}
}

// Init 初始化
func (p *Processor) Init(config []byte) error {
	if err := json.Unmarshal(config, p.config); err != nil {
		return fmt.Errorf("解析配置失败: %w", err)
	}
	if p.config.Field == "" {
		return fmt.Errorf("field 不能为空")
	}
	if p.config.Operator == "" {
		p.config.Operator = "eq" // 默认操作符
	}
	return nil
}

// Metrics 声明自定义指标（步骤1：必须实现）
func (p *Processor) Metrics() []dataflow.MetricDesc {
	return []dataflow.MetricDesc{
		{
			Name:   "filter_passed_total",
			Type:   dataflow.MetricTypeCounter,
			Help:   "Total number of records that passed the filter",
			Labels: []string{"flow", "component", "field", "operator"},
		},
		{
			Name:   "filter_dropped_total",
			Type:   dataflow.MetricTypeCounter,
			Help:   "Total number of records dropped by the filter",
			Labels: []string{"flow", "component", "field", "operator"},
		},
	}
}

// SetMetricsRecorder 框架注入指标记录器（步骤2）
func (p *Processor) SetMetricsRecorder(recorder dataflow.MetricsRecorder, info dataflow.ComponentInfo) {
	p.recorder = recorder
	p.info = info
}

// Process 处理数据（步骤3：使用指标）
func (p *Processor) Process(ctx context.Context, in <-chan types.Record, out chan<- types.Record) error {
	// 准备标签（业务相关标签）
	baseLabels := prometheus.Labels{
		"field":    p.config.Field,
		"operator": p.config.Operator,
	}

	for record := range in {
		select {
		case <-ctx.Done():
			return ctx.Err()
		default:
		}

		if p.match(record) {
			// 记录通过过滤的数量
			p.recorder.AddCounter("filter_passed_total", 1, baseLabels)
			out <- record
		} else {
			// 记录被丢弃的数量
			p.recorder.AddCounter("filter_dropped_total", 1, baseLabels)
		}
	}
	return nil
}

// match 判断记录是否匹配过滤条件
func (p *Processor) match(record types.Record) bool {
	value, ok := record[p.config.Field]
	if !ok {
		return false
	}

	switch p.config.Operator {
	case "eq":
		return fmt.Sprintf("%v", value) == fmt.Sprintf("%v", p.config.Value)
	case "ne":
		return fmt.Sprintf("%v", value) != fmt.Sprintf("%v", p.config.Value)
	case "contains":
		return contains(fmt.Sprintf("%v", value), fmt.Sprintf("%v", p.config.Value))
	default:
		return false
	}
}

func contains(s, substr string) bool {
	return len(s) >= len(substr) && s[:len(substr)] == substr
}
```

启用 metrics 后，可以在 Prometheus 中查询：


### 8.6 关键点总结

1. **声明指标**: 实现 `Metrics() []MetricDesc` 声明两个 Counter 指标
2. **注入记录器**: 实现 `SetMetricsRecorder(recorder, info)` 接收框架注入
3. **记录指标**: 在 `Process` 中调用 `AddCounter` 记录业务指标
4. **标签处理**: 只传入业务标签（`field`, `operator`），框架自动附加 `flow` 和 `component`

## 9. 开发清单

开发自定义组件时，确保：

- [ ] 定义配置结构体，包含 JSON 标签
- [ ] 配置有合理的默认值
- [ ] Init 中校验必填字段
- [ ] Read/Process/Consume 支持 ctx 取消
- [ ] 使用 select 防止通道阻塞
- [ ] 错误信息包含上下文
- [ ] 在 main.go 中注册组件
- [ ] （可选）实现 `MetricsRecorderAware` 接口，使用 `MetricsRecorder` 记录自定义业务指标
- [ ] （可选）在 app.yaml 中启用 metrics 服务进行监控
