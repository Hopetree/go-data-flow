# 租户二次开发指南

本文档指导如何基于 go-data-flow 框架导入、扩展和部署自定义组件。

## 1. 概述

go-data-flow 以 Go module 形式发布，租户通过 `go.mod` 依赖引入框架，在自有项目中注册内置组件和自定义组件。框架与租户代码完全解耦，互不影响。

```
Tenant Project                   go-data-flow Framework
+--------------------+           +---------------------------+
| cmd/myapp/main.go  |---------> | pkg/dataflow/app/         |
|                    |           |   builtins.go             |
| components/        |           |                           |
|   mysource/        |           | pkg/dataflow/builtins/    |
|   mysink/          |           |   source/ processor/ sink/|
+--------------------+           +---------------------------+
     go.mod:                     module:
     require go-data-flow        github.com/Hopetree/go-data-flow
```

## 2. 创建项目

### 2.1 初始化

```bash
mkdir my-dataflow && cd my-dataflow
go mod init my-dataflow
```

### 2.2 引入框架

```bash
go get github.com/Hopetree/go-data-flow@latest
```

### 2.3 项目结构

建议采用以下标准化目录结构：

```
my-dataflow/
├── cmd/myapp/
│   └── main.go              # 入口，注册组件并启动
├── internal/
│   ├── components/          # 自定义组件目录
│   │   ├── source/          # 自定义 Source
│   │   │   └── reader.go    # 自定义组件，根据实际需要创建，初始留空
│   │   ├── processor/       # 自定义 Processor
│   │   └── sink/            # 自定义 Sink
│   │       └── writer.go    # 自定义组件，根据实际需要创建，初始留空
│   └── others/              # 其他自定义模块目录（API 请求、组件公共依赖等，目录名称根据实际创建），初始留空
├── conf/                    # 配置文件
│   ├── app.yaml             # 应用全局配置
│   └── flows/               # Flow 配置文件目录
│       └── example.yaml
├── scripts/
│   └── build.sh             # 跨平台打包脚本
├── go.mod
└── go.sum
```

> 项目创建完成后，需要同步创建打包脚本 `scripts/build.sh`，可直接参考框架项目的 [build.sh](../../scripts/build.sh) 实现，修改 `BINARY_NAME` 和 `MAIN_PATH` 即可。

## 3. 编写入口

`main.go` 是租户项目的核心，负责注册内置组件和自定义组件：

```go
package main

import (
	"context"
	"flag"
	"os"

	"github.com/Hopetree/go-data-flow/pkg/dataflow/app"
	"github.com/Hopetree/go-data-flow/pkg/logger"
	// 引入租户自定义组件
	"my-dataflow/internal/components/source"
	"my-dataflow/internal/components/sink"
)

// 构建时注入的版本信息（配合 build.sh -ldflags 使用）
var (
	Version   = "dev"
	BuildTime = "unknown"
	GitCommit = "unknown"
)

// 命令行参数
var (
	appConfFile string
	configFile  string
	configDir   string
	configs     string
	listOnly    bool
	showVersion bool
)

func init() {
	flag.StringVar(&appConfFile, "a", "", "应用配置文件路径 (conf/app.yaml)")
	flag.StringVar(&configFile, "c", "", "Flow 配置文件路径")
	flag.StringVar(&configDir, "d", "", "配置文件目录")
	flag.StringVar(&configs, "C", "", "多个配置文件，逗号分隔")
	flag.BoolVar(&listOnly, "l", false, "列出所有组件")
	flag.BoolVar(&showVersion, "v", false, "显示版本")
}

func main() {
	flag.Parse()

	// 设置版本信息
	app.Version = Version
	app.BuildTime = BuildTime
	app.GitCommit = GitCommit

	// 显示版本
	if showVersion {
		app.PrintVersion()
		return
	}

	// 创建应用
	application := app.NewApp(app.Options{
		AppConfFile: appConfFile,
		ConfigFile:  configFile,
		ConfigDir:   configDir,
		Configs:     configs,
	})

	// 注册内置组件
	app.RegisterAllBuiltins(application.Registry())

	// 注册自定义组件（每个组件包提供 Register 函数）
	source.Register(application.Registry())
	sink.Register(application.Registry())

	// 列出组件
	if listOnly {
		application.ListComponents()
		return
	}

	// 检查是否有配置
	if configFile == "" && configDir == "" && configs == "" {
		app.PrintUsage()
		os.Exit(1)
	}

	// 创建上下文
	ctx := context.Background()

	// 运行应用
	if err := application.Run(ctx); err != nil {
		logger.Error("运行失败: %v", err)
		os.Exit(1)
	}
}
```

### 3.1 构建和运行

```bash
# 构建
go build -o myapp ./cmd/myapp

# 运行
./myapp -a conf/app.yaml -c conf/flows/example.yaml

# 列出所有组件（内置 + 自定义）

```

## 4. 开发自定义组件

自定义组件实现框架定义的接口，通过 `Registry` 注册即可使用。详细接口定义参见 [自定义组件开发指南](01_自定义组件开发指南.md)。

### 4.1 组件注册模式

每个自定义组件包应提供一个 `Register` 函数，接受 `Registry` 参数：

```go
// internal/components/source/reader.go
package source

import (
    "github.com/Hopetree/go-data-flow/pkg/dataflow"
    "github.com/Hopetree/go-data-flow/pkg/dataflow/builtins/types"
)

const TypeName = "source-my-data"

// MySource 自定义 Source 组件
type MySource struct {
    filePath string
}

func New() *MySource {
    return &MySource{}
}

func (s *MySource) Init(config []byte) error {
    // 解析配置
    return nil
}

func (s *MySource) Read(ctx context.Context, out chan<- types.Record) (int64, error) {
    // 读取数据并发送到通道
    return 0, nil
}

// Register 注册组件到 Registry
func Register(r *dataflow.Registry[types.Record]) {
    r.RegisterSource(TypeName, func() dataflow.Source[types.Record] {
        return New()
    })
}
```

### 4.2 在 Flow 配置中使用

```yaml
# conf/flows/example.yaml
name: my-etl-flow
buffer_size: 100

source:
  name: source-my-data          # 自定义组件名
  config:
    file_path: ./data/input.json

processors:
  - name: processor-transform-field  # 内置组件
    config:
      add:
        processed: true

sink:
  name: sink-json-file             # 内置组件
  config:
    file_path: ./output/result.jsonl
```

### 4.3 可选接口

组件可按需实现以下接口获得额外能力：

| 接口 | 方法 | 说明 |
|------|------|------|
| `Closer` | `Close() error` | Flow 结束后清理资源 |
| `ConcurrencyCapable` | `Concurrency() int` | 声明并发 worker 数 |
| `MetricsCapable` | `RecordMetrics(recorder)` | 自定义 Prometheus 指标 |

### 4.4 复用内置基类

Processor 组件可直接嵌入框架提供的基类，避免重复实现并发控制：

```go
import (
    "github.com/Hopetree/go-data-flow/pkg/dataflow/builtins/processor"
    "github.com/Hopetree/go-data-flow/pkg/dataflow/builtins/types"
)

// 单协程顺序处理
type MyProcessor struct {
    processor.BaseProcessor[types.Record]
}

// 无状态并发处理（每条记录独立）
type MyConcurrentProcessor struct {
    processor.StatelessProcessor[types.Record]
}
```

## 5. 框架升级

```bash
# 查看可升级版本
go list -m -versions github.com/Hopetree/go-data-flow

# 升级到最新版本
go get -u github.com/Hopetree/go-data-flow

# 升级到指定版本
go get github.com/Hopetree/go-data-flow@v0.2.0

# 整理依赖
go mod tidy

# 验证编译
go build ./...
```

由于框架与租户代码完全解耦，升级不会影响自定义组件的实现。如果框架接口有破坏性变更，会在 Release Notes 中说明。

## 6. 多租户组件共享

如果团队内有多个项目需要共享同一套自定义组件，可以将组件抽取为独立的 Go module：

```
github.com/yourorg/dataflow-components/
├── go.mod
├── clickhouse_reader/
│   └── reader.go
├── mysql_source/
│   └── source.go
└── ...
```

在各项目中引用：

```bash
go get github.com/yourorg/dataflow-components@latest
```

```go
import (
    _ "github.com/yourorg/dataflow-components/clickhouse_reader"
    _ "github.com/yourorg/dataflow-components/mysql_source"
)
```
